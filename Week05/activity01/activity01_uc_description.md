### Use Case Diagram: Restaurant Ingredient Ordering 

## Purpose: 
Manage ingredient inventory and procurement

## Actors: 
Ingredient Supply Manager, Restaurant Manager 

## Use Case 1 - Create Order: 
Ingredient Supply Manager creates an order to buy ingredients according to the necessity and submit to the system.

## Use Case 2 & 3 - Review Order + Approve and Send Order: 
Restaurant Manager reviews the order submitted by the Ingredient Supply Manager. If the order is justifiable, then the Restaurant Manager will dispatch an order to the vendor. 

## Use Case 4 - Track Order Status: 
Ingredient Supply Manager does the follow-up check about the order request he submitted. Track Order Status is an extending functionality of Send Order, so Restaurant Manager can do the follow-upp check as well to know the status of the delivery of the ordered ingredients from the vendor.
